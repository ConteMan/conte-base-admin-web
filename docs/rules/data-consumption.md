# 数据消费与元数据入口规则

## 开始前先判定数据层

涉及枚举、下拉、状态、配置项、参考数据时，先判断属于哪一层：

1. 代码枚举
2. 只读参考数据
3. 运营数据字典
4. 系统配置 / Feature Flag
5. 独立业务实体

## 前端消费入口

根据数据层选择合适入口：

- `useEnum`
- `useLookup`
- `useDict`
- `useConfig`
- `useFeatureFlag`

## 约束

- 独立业务模块应优先走模块专属 API，不反向依赖通用元数据入口。
- 不要把状态机枚举、权限码或协议常量降级到运营字典。

## API 对接

- 具体接口对接前，先确认需求文档，再确认 API 协议规范，最后读取 Swagger / OpenAPI 字段契约。
- 路径、参数、Request / Response Schema 与 DTO 以 Swagger 为准。
- 业务语义、权限、fallback、留痕与本期范围以需求文档为准。
- 不允许凭接口命名、页面历史代码或口头约定直接编写请求。
